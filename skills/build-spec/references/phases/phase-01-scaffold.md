# Phase 1: Scaffold

<objective>
Create project infrastructure from spec.yaml that is NOT component code. Then run baseline /opensdd:compare.
</objective>

<prerequisite>
Phase 0 must be complete (prerequisites verified).
</prerequisite>

<input>

From spec.yaml:
- `tech_stack`: Language, framework, dependencies
- `structure`: Directory layout, entrypoints, tests path
- `types`: Shared type definitions
- `deployment`: Deployment target and configuration (optional)
</input>

<steps>

<step n="1" name="read_spec">
Load spec.yaml, show summary, and identify sections with deliverables:

```bash
cat .opensdd/spec.yaml
```

Display build summary:
```
Build Spec: Starting
════════════════════

Language: {tech_stack.language}
Components: {count of components}
Types: {count of types}

Components to implement:
  - {component_name} ({layer})
  - ...
```

Identify what needs scaffolding:
- `tech_stack` → project config file
- `structure.layers` → directories
- `structure.entrypoints` → entry point files
- `types` → type definition files
- `deployment` → deployment files (if present)
- `structure.tests` → test directory (if present)
</step>

<step n="2" name="scaffold_structure">
Create directory structure from `structure.layers`.

For each layer, create the directory and any *language-required* module files (e.g., `__init__.py` for Python).
</step>

<step n="3" name="scaffold_project_config">
Generate project configuration file based on `tech_stack.language`.

Skip if file already exists. Include dependencies from `tech_stack.dependencies`.
</step>

<step n="4" name="scaffold_entrypoints">
Create entry point stubs from `structure.entrypoints`.

For each entrypoint, create a minimal valid stub if it doesn't exist.
</step>

<step n="5" name="scaffold_types">
Create type definitions from `types:` section.

**Philosophy:** Spec defines what types exist. Fields are added during component implementation.

For each type in spec:
- If has `enum:` field → create complete enum
- Otherwise → create skeleton with docstring from `for:` field

Skip types that already exist in file.
</step>

<step n="6" name="scaffold_deployment">
If `deployment` section exists, create deployment files based on target. Skip if files already exist.
</step>

<step n="7" name="scaffold_tests">
If `structure.tests` defined, create test directory and config file.
</step>

<step n="8" name="scaffold_readme">
Create minimal README.md if it doesn't exist:

```markdown
# {project_name from spec or directory name}

{spec.overview if present, otherwise one-line description}

## Tech Stack

- **Language:** {tech_stack.language}
- **Framework:** {tech_stack.framework}

## Getting Started

{Basic setup instructions based on tech_stack.language}

## Project Structure

{List structure.layers with brief descriptions}

---
*Generated by OpenSDD*
```

Skip if README.md already exists.
</step>

<step n="9" name="update_gitignore">
Add OpenSDD patterns to .gitignore:

```bash
if ! grep -q "# OpenSDD" .gitignore 2>/dev/null; then
  cat >> .gitignore << 'EOF'

# OpenSDD - build artifacts (regeneratable)
.opensdd/extracted/
.opensdd/compare-result.yaml
EOF
  echo "Added OpenSDD patterns to .gitignore"
else
  echo "OpenSDD patterns already in .gitignore"
fi
```
</step>

<step n="10" name="run_baseline_compare">
Run /opensdd:compare to establish baseline:

```
/opensdd:compare
```

This creates `.opensdd/compare-result.yaml` showing:
- All components as "missing" (nothing implemented yet)
- Types status (if tracked)

**Schema:** See `skills/compare-spec/references/output-schema.yaml` for compare-result.yaml structure.

Read the result:
```bash
cat .opensdd/compare-result.yaml
```

Display baseline summary:
```
Scaffold Complete
═════════════════

Created:
  - Directory structure
  - Project config
  - Entry points
  - Type skeletons
  - README.md
  - {other items}

Baseline Compare:
  Components: {total} (all missing - ready to implement)

Starting component implementation...
```
</step>

</steps>
